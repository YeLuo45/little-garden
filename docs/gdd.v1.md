### 3. 角色设计风格

#### 3.1 角色设计总则

- **体型**：圆滚滚、矮胖可爱，头身比约 1:1.2~1:1.5（大头娃娃风格）
- **线条**：轮廓线使用 2-4px 粗的深棕色线（#5D4037），圆润无锐角
- **眼睛**：占比大（占脸部1/3以上），高光点明显，表达单纯可爱的情感
- **嘴巴**：小巧圆润，笑起来是自然的 U 形弧线
- **四肢**：短小圆润，手掌脚掌是圆润的椭圆/圆形
- **阴影**：统一使用底部软阴影（椭圆形），不画复杂光影
- **表情**：所有角色都有 idle 待机表情循环（呼吸感眨眼）

#### 3.5 角色设计模板规格

```
单角色 sprite 规格：
- 基础待机帧：1帧（idle循环通过代码实现scale/opacity）
- 行走帧：4帧（左右各2帧，或四面八方各1帧）
- 动作帧（举手/跳跃/掉落）：2-4帧
- 表情帧（开心/惊讶/错误）：2帧
- 单帧建议尺寸：128x128px / 256x256px（圆整到2^n）

Canvas渲染时使用整数坐标，保持像素清晰
禁止使用复杂的骨骼动画，全部使用 sprite sheet 或程序化动画
```

### 4. 场景美术方向

#### 4.1 整体场景风格

- **美术风格**：2D平面视角 + 轻微透视感（俯视 15°~30°），类似绘本插画
- **空间感**：场景有明确的层次感（远景/中景/近景），层次间有轻微视差滚动
- **边缘处理**：所有物体边缘圆润，避免锯齿和锐利边角
- **地面**：统一使用无缝平铺草地纹理（程序化生成可平铺的草地图案）
- **天空**：渐变色背景 + 装饰性云朵和太阳/月亮

#### 4.2 主场景：花园地图

```
[远景层] 淡蓝天空 + 白色云朵 + 太阳/月亮（根据时间）
     ↓
[中景层] 远树剪影、远处小山丘、装饰性风车/篱笆
     ↓
[近景层] 可交互的花园场景
  ├── 草地（程序化生成的绿色地面）
  ├── 花坛（各种颜色的花朵，幼儿模式有更多颜色）
  ├── 树木（苹果树、向日葵等）
  ├── 池塘（带有荷叶和小鱼）
  ├── 小路（引导玩家移动）
  └── 可点击的NPC/物体

[UI覆盖层] 顶部星星计数、模式切换按钮、暂停按钮
```

### 5. UI视觉规范

#### 5.1 UI设计总则

- **风格**：圆润卡片式设计，所有按钮和面板使用圆角矩形（border-radius: 16-24px）
- **字体**：使用粗圆体/童趣字体（如 Noto Sans SC Black 或系统圆体 fallback）
- **字号层级**：
  - 标题：32-40px（幼儿模式加大10px）
  - 正文：24-28px
  - 辅助：18-20px
- **按钮最小尺寸**：
  - 幼儿模式：80x80px（核心操作），最小不低于 60x60px
  - 儿童模式：60x60px（核心操作），最小不低于 48x48px
- **间距**：元素间距最小 16px，确保触控不误触
- **阴影**：卡片使用柔和投影 `shadow: 0 4px 12px rgba(0,0,0,0.15)`

#### 5.2 主要UI组件规范

##### 按钮（Button）

```
默认状态：
  - 背景：#4CAF50（主操作）/ #81C784（次操作）
  - 圆角：20px
  - 文字：白色，粗体，24px
  - 内边距：16px 32px
  - 轻微3D感（下边框深色 4px + 主体）

悬停/按下状态（触摸屏为触摸时）：
  - 整体向下位移2px
  - 下边框从4px减少到2px
  - 背景色加深10%

禁用状态：
  - 背景：#BDBDBD
  - 文字：#757575
  - 无投影

幼儿模式按钮额外规范：
  - 必须带图标（图标 > 文字）
  - 按钮周围有粗边框（3px #5D4037）
  - 按钮带有微妙的摇晃/弹跳动画
```

##### 卡片（Card）

```
背景：#FFF8E1（暖白）或白色
圆角：24px
边框：无（靠阴影区分）
阴影：0 4px 16px rgba(0,0,0,0.12)
内边距：24px
```

##### 进度条/生命值

```
背景轨道：#E0E0E0（灰色圆角条）
填充条：渐变 #66BB6A → #43A047
圆角：12px
高度：20px（幼儿模式 28px）
动画：填充变化时使用 ease-out 动画 300ms
```

##### 弹窗/对话框

```
背景：白色圆角面板
边框：3px 可爱图案边框（如小花朵边框）
阴影：大阴影 0 8px 32px rgba(0,0,0,0.2)
关闭按钮：右上角大 X（幼儿模式要求）
```

##### 星星/奖励显示

```
星星图标：5角星形状，#FFD700 填充，橙色边框
尺寸：32x32px（幼儿模式 48x48px）
动画：获得时弹跳放大 1.2x + 发光 + 旋转
累计显示：使用数字 + 星星图标组合
```

### 6. 动画风格

#### 6.1 动画设计总则

- **风格**：弹性动画（Elastic / Bouncy），所有物体运动带有自然回弹
- **帧率**：目标 60fps，使用 Canvas requestAnimationFrame
- **时长**：
  - 微交互（按钮悬停）：150-200ms
  - 状态切换（打开/关闭）：300-400ms
  - 剧情/奖励动画：800-1500ms
  - 角色待机循环：2000-3000ms 循环
- **缓动函数**：统一使用 ease-out 或 elastic-out，禁止 linear

#### 6.2 各类动画规范

##### 角色动画

```
待机（Idle）：
  - 轻微上下浮动（Y轴 ±3px）
  - 每3秒一次眨眼动画（眼睛 scaleY 0.1 → 1）
  - 头发/配件轻微摇摆

移动（Walk）：
  - 左右移动时身体轻微左右摇摆（rotation ±5°）
  - 腿部/脚掌交替迈步（通过 scaleX 实现简易动画）
  - 速度：200px/秒

跳跃（Jump）：
  - 起跳：快速向上（ease-out，150ms）
  - 空中：身体轻微压缩（scaleY 0.9）
  - 落地：弹性回弹（elastic-out，200ms）
  - 高度：约 40-60px

庆祝（Celebrate）：
  - 双手举起 + 身体上下弹跳
  - 眼睛变成 ^ ^ 形状（笑眼）
  - 配合星星/花瓣粒子效果
```

##### 反馈动画

```
正确反馈：
  - 绿色对勾图标弹出（scale 0 → 1.2 → 1，300ms）
  - 目标元素变绿 + 发光
  - 星星飞向计数器
  - 音效：清脆的"叮"声

错误反馈：
  - 红色叉号弹出（shake 抖动，300ms）
  - 目标元素轻微摇晃
  - 屏幕边缘红色闪一下
  - 音效：柔和的"咚"声（不刺耳）

奖励结算：
  - 星星从各方飞入汇总
  - 大星星居中放大 + 光芒扩散
  - 角色跳舞庆祝动画
```

##### UI动画

```
按钮按下：
  - scale 1 → 0.95 → 1
  - duration: 150ms
  - ease: ease-out

弹窗出现：
  - scale 0.8 → 1 + opacity 0 → 1
  - duration: 300ms
  - ease: elastic-out

页面切换：
  - 当前页向左滑出 + 淡出
  - 新页从右滑入 + 淡入
  - duration: 400ms
  - ease: ease-out

视差滚动（场景层）：
  - 远景层移动速度：0.3x
  - 中景层移动速度：0.6x
  - 近景层移动速度：1x
```

### 8. 技术实现注意事项

#### 8.1 Canvas 渲染规范

- **分辨率**：基础 1280x720，支持缩放适配全屏
- **坐标系统**：统一使用整数坐标，避免亚像素渲染导致模糊
- **层级管理**：使用 z-index 分层（背景/角色/UI/特效）
- **缓存策略**：静态背景使用离屏 Canvas 缓存，避免每帧重绘

#### 8.2 资源加载策略

- **图片格式**：PNG（带透明）+ WebP（渐进式加载）
- **Sprite Sheet**：将同角色/同场景资源合并为 Atlas，减少请求数
- **按需加载**：非当前场景资源延迟加载
- **进度显示**：加载过程显示进度条 + 小园宝动画

#### 8.3 可访问性注意事项

- 所有颜色组合需满足 WCAG AA 对比度（文字 vs 背景 ≥ 4.5:1）
- 不依赖颜色传达信息（同时使用图标/形状辅助）
- 动画可关闭（尊重减少动画偏好设置）
- 大触控区域（符合儿童友好标准）
